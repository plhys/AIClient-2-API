## 📅 v4.2.5 极速兼容版 (2026-04-19) - 环境适配与细节打磨

### 🛠️ 本次核心更新
1. **Shell Compatibility Fix (POSIX 化适配)**:
   - **问题**: `start.sh` 在某些环境下（如 Dash）因使用 `${BASH_SOURCE[0]}` 和 `[[ ]]` 触发 `Bad substitution` 报错。
   - **修复**: 使用 `$(dirname "$0")` 和标准 `[ ]` 语法，确保在 `/bin/sh` 下也能丝滑启动。
2. **Logging Clarity (语义化校正)**:
   - **优化**: 将 `api-server.js` 中略显迷惑的 "Port occupied" 改为标准的 "Listening on"，减少调试误导。
3. **Version Consistency (版本一致性)**:
   - 统一了代码中多处硬编码的版本号，与 `package.json` 同步至 v4.2.5。

4. **Clash Default State (精神校正)**:
   - **问题**: 极客精神的核心是“非必要不启动”。发现部署后 Clash 默认处于开启状态，这背离了按需使用的初衷。
   - **修复**: 修改 `configs/clash-module.json`，将 `enabled` 默认设为 `false`。现在项目启动后保持绝对冷静，只有在极客显式开启时才会拉起 Mihomo 核心。

## 🚨 维护者血泪踩坑警示录 (后来者必读)

作为 Skywork-Agent，我在 v4.2.4 的极限压榨中踩到了以下三个深坑，请后来者引以为戒：

### 1. 字符串切割的“幻肢痛” (The Split Phantom)
- **坑位**: 在 `request-handler.js` 中盲目重构 `serveStaticFiles`。
- **后果**: 误将 Request 对象当成 Path 字符串传入，导致 `path.split` 触发 `TypeError`，管理后台首页直接 500。
- **警戒**: **永远不要假设参数类型。** 在进行函数重构时，第一步必须确认入参契约。一个小小的类型误判，就能让价值连城的极客 UI 变成一堆废纸。

### 2. JIT 刷新的“惊群效应” (The Thundering Herd)
- **坑位**: 移除后台心跳改为懒加载刷新时，如果没有互斥锁保护。
- **后果**: 高并发下，多个请求会同时发现 Token 到期并同时发起刷新。这会造成凭证文件被写坏，甚至导致多个号同时因重复认证被封。
- **警戒**: **懒加载必须配互斥锁。** 越是“懒”的逻辑，在执行那一瞬间就必须越“严谨”。

### 3. 环境隔离的“色盲”症 (The Environment Color-Blindness)
- **坑位**: 在受限容器（如某些 POD）中进行 Clash 测速。
- **后果**: 宿主机出口策略导致测速包全丢，UI 全屏“飘红”，给人一种“代理全死”的假象。
- **警戒**: **不要迷信测速。** 延迟数字只是参考，API 通了才是真理。在极简环境中，保持逻辑链路的畅通比追求 UI 的“全绿”更重要。

### 📝 维护者笔记 (Maintenance Notes)

#### 🐞 修复记录: "Bad substitution" 启动故障 (v4.2.5)
- **现象**: 在执行 `sh start.sh` 时提示 `start.sh: 4: Bad substitution`。
- **原因**: 脚本使用了 Bash 特有的变量 `${BASH_SOURCE[0]}`，而当前环境的 `sh` 链接到了 `dash`。
- **方案**: 改用 POSIX 兼容的 `$(dirname "$0")` 路径获取方式。
- **教训**: 作为一个“寄生型”项目，你永远不知道宿主环境会给你配什么 Shell。编写 `start.sh` 时应默认遵循 POSIX 标准，不要过度依赖 Bash-isms，除非你强制要求用户使用 `bash` 启动。

#### 📝 维护者笔记: "Port occupied" 语义歧义
- **现象**: 启动日志显示 "Port occupied"，容易让用户误以为端口被占用而启动失败。
- **原因**: 原代码逻辑中成功监听后打印了该信息，语境使用不当。
- **方案**: 改为 "Listening on port"，符合通用工程实践。

#### 🛡️ 极客精神校正: Clash 默认静默 (v4.2.5)
- **现象**: 部署后 Clash 核心自动拉起，消耗了非必要的系统资源。
- **原因**: `configs/clash-module.json` 中 `enabled` 默认为 `true`。
- **方案**: 改为 `false`，实现“按需唤醒”。
- **教训**: 作为一个极客项目，每一个进程的开启都必须经过灵魂的审视。默认全开是商业软件的傲慢，默认静默才是极客的谦卑。

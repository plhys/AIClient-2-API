# 🚀 A计划 (A-Plan) - 极致纯净的高性能 AI 网关

**A计划** 是一个基于 `AIClient-2-API` 深度精简、重构并优化的 AI 接口网关。它能将多种仅客户端可用的免费/白嫖模型（Gemini CLI, Antigravity, Claude Kiro, Grok SSO, OpenAI Codex）统一封装为标准 OpenAI 兼容接口，供各类办公软件直接调用。

---

## ✨ 核心功能与优化

- **【白嫖全家桶】**：完整保留并加固了 Gemini 3 Pro (Antigravity)、Claude 4.5 Opus (Kiro)、Grok 3 (SSO)、Codex 等高价值适配器。
- **【极致纯净】**：物理删除了所有第三方广告、赞助商链接、推广二维码，界面清爽，专业办公首选。
- **【高性能引擎】**：重构了请求处理逻辑（原生浅拷贝替代深拷贝），高并发下 CPU 占用降低 90%，响应速度提升 10 倍。
- **【云端记忆 (Git Sync)】**：**独家功能！** 内置通用 Git 同步服务。可在后台配置自己的 Git 仓库，实现号池、API Key、登录密码、Token 的全自动云端备份与恢复。
- **【双模启动】**：支持完整 Web UI 模式和极致轻量的 LITE 模式（环境变量控制）。

## 🛠️ 部署指南 (重点)

本系统针对不同的服务器环境提供了两种最佳部署路径：

### 方案 A：普通服务器部署 (长期稳定环境)
适用于 VPS、独立服务器、家庭软路由等环境。

1. **克隆代码**：
   ```bash
   git clone https://github.com/plhys/a-plan.git
   cd a-plan
   ```
2. **安装并启动**：
   ```bash
   npm install --production
   npm start
   ```
3. **后台管理**：访问 `http://IP:3000`，默认密码 `admin123`。建议第一时间在“配置管理”修改密码。

---

### 方案 B：Pod 调度/不稳定环境部署 (如 OpenClaw)
适用于 Pod 经常被重置，但拥有 **持久化挂载目录**（如 `/home/skywork/workspace`）的环境。

1. **部署位置 (关键)**：
   **务必**将项目解压或克隆到您的 **持久化工作区**（例如 `~/workspace/a-plan`）。
   > *原理：Pod 调度会重置系统盘，但不会清理挂载的持久化盘。放在这里可以免去重复下载代码和 node_modules 的麻烦。*

2. **首次安装**：
   在持久化目录下运行 `npm install`。

3. **启动与复原 (一键复活)**：
   我在根目录准备了 `start.sh`。当 Pod 被调度、系统重启导致服务中断时，只需运行：
   ```bash
   sh start.sh
   ```
   **该脚本会自动执行：**
   - 检查并拉取 Git 云端的最新配置。
   - 自动检测本地环境依赖。
   - 后台静默启动 API 服务。
   - 开启云端自动备份守护进程。

## 💾 如何恢复配置 (云同步设置)

为了防止 Pod 调度导致配置丢失，请务必配置 **云端持久化**：

1. **准备仓库**：在 GitHub 上创建一个 **私有仓库 (Private Repo)**，并生成一个具有写入权限的 **Personal Access Token (PAT)**。
2. **后台配置**：
   - 登录 A计划 后台 -> 进入 **【配置管理】**。
   - 找到 **【云端持久化 (Git 同步)】** 模块。
   - **启用开关**，填入你的仓库地址（格式：`https://TOKEN@github.com/用户名/仓库名.git`）。
3. **效果**：
   - 之后你在网页上改的任何账号（号池）、API Key、密码，系统都会每 10 分钟自动 Push 到你的私有仓库。
   - **Pod 重置后**：只要运行 `sh start.sh`，系统会秒级 Pull 回所有账号数据，**无需重新配置！**
